# Daily Compass

Codexとの相談で決めた方針をもとに、毎日の3クエストと今週のクエストを表示するローカルWebアプリです。

## 開き方

`index.html` をブラウザで開いてください。ローカルサーバーを使う場合は `python serve.py --port 8788` です。PCでは `http://127.0.0.1:8788/`、スマホでは起動時に表示される `Phone:` のURLを開いてください。スマホとPCは同じWi-Fi/LANに接続してください。開けない場合は、WindowsファイアウォールでPythonの通信を許可してください。

`192.168...` のURLは自宅や同じWi-Fi内だけで使えるローカルURLです。外出先のスマホ回線から直接開くには、公開URL化またはクラウド配置が必要です。暫定的に見るだけなら、今日画面の「外出用メモ」で今日/今週のクエストをコピーして持ち出せます。

クラウド版として動かす場合は `DAILY_COMPASS_PASSWORD` を設定して `python serve.py --host 0.0.0.0` で起動します。設定するとログイン画面が有効になり、アプリ本体と `/api/state` が保護されます。外出先スマホ利用の実行手順は `CLOUD_DEPLOY_NOW.md`、詳細な配置メモは `DEPLOYMENT.md` を参照してください。

進捗データはブラウザの `localStorage` に保存されます。Codexが更新する標準ルールは `quest-rules.js` に分けてあるため、通常のルール変更で完了履歴は消えません。
ローカルサーバーで開いた場合は `app-state.json` にも自動バックアップされます。スマホ用URLのIPが変わってブラウザ保存が空に見える場合でも、サーバー側の状態から復元しやすくなります。

## 保存領域

- `daily-compass-progress-v2`: 完了、入れ替え、デイリー/ウィークリーの記録
- `daily-compass-schedule-v2`: 勤務日、休み、代休などの予定変更
- `daily-compass-custom-quests-v1`: アプリ内またはCodex JSONで追加したクエスト候補
- `daily-compass-quest-inbox-v1`: Codexに渡すための連携メモ
- `daily-compass-quest-feedback-v1`: 出したくないクエストとその理由
- `daily-compass-quest-tuning-v1`: 重さ、想定時間、頻度の調整内容
- `app-state.json`: ローカルサーバー側の自動バックアップ

EXP、称号、ご褒美は完了履歴から自動計算されるため、専用の保存領域や手入力はありません。

クラウド版では `DAILY_COMPASS_STATE_FILE` で保存先を指定できます。永続ディスクを使う場合は、例として `/var/data/app-state.json` を指定します。

## 更新時の安全ルール

このアプリで最優先することは、アプリ内で積み上げた調整と、Codexから反映する修正をどちらも失わないことです。

- Codexがコードを修正しても、`localStorage` と `app-state.json` の保存データを初期化しない
- 標準ルールの修正は主に `quest-rules.js` で行い、完了履歴やアプリ内調整とは分ける
- アプリで追加したクエスト、除外理由、重さ/時間/頻度の調整はユーザーデータとして扱い、コード更新で上書きしない
- Codexからのクエスト追加、除外、調整はJSON取り込みか既存保存領域への追記として扱う
- 同じクエストIDに対する修正は、既存データを消すのではなく、差分更新または新しい調整として残す
- 履歴は追記型を基本にし、完了履歴を削除・再生成する変更は避ける
- 大きな構成変更やクラウド化の前には、履歴画面のエクスポートと `app-state.json` をバックアップする

クラウド版にする場合も、同じ境界を守ります。アプリ本体、標準ルール、ユーザーの進捗、アプリ内調整、Codexからの反映内容を分けて保存し、デプロイやコード修正で個人データがゼロに戻らない構成にします。

## できること

- 今日の3クエストをゲーム風の掲示板として表示
- 完了と入れ替えだけで操作
- 入れ替えても今日の3枠を維持
- 詳細ボタンで分類、重さ、判断、登録元を確認し、重さ/想定時間/頻度を調整
- 勤務日、月曜休み、育児あり休日、特別休みを月間カレンダーで設定
- 火曜日から月曜日の今週クエスト3つを1回チェックで管理
- 各クエストカードに獲得EXPを表示
- 完了履歴からEXP、称号、次のご褒美を自動表示
- 今日/今週のクエスト、EXP、ご褒美を外出用メモとしてコピー
- 称号とご褒美は現在地、次の目標、少し先の予告を表示
- 目的を逆に押さない範囲で、本人が喜ぶご褒美案とゲーム感覚で続けるコツを表示
- ご褒美は一人時間、休み時間、銭湯、別ゲーム、動画/アニメを中心にし、夜食や爆食いは避ける
- 日付とEXPに応じた「Codexからの贈り物」で、自分では思いつきにくいご褒美候補を表示
- 調整画面で標準候補を含む全クエストの詳細を開き、重さ/想定時間/頻度/メモを調整
- 調整画面で分類、重さ、頻度、出してよい日を指定してクエスト候補を追加
- 調整画面で既存クエストを理由つきで除外し、必要なら復帰
- Codex用データの作成と、Codexからのクエスト/除外JSON取り込み
- ルール画面で現在の考え方と候補プールを可視化
- 履歴と進捗データをエクスポート
- エクスポートしたバックアップJSONを取り込み、クラウド初回移行や復旧に使う

## Codexからクエストを取り込む形式

調整画面の「CodexからのJSON」には、次のような形式を貼り付けられます。

```json
{
  "quests": [
    {
      "title": "配信後に次回使うポケモン候補を1体メモする",
      "categories": ["youtube", "pokemon"],
      "weight": "light",
      "minutes": 10,
      "frequency": "medium",
      "dayFit": ["work", "mondayRest", "childcareRest", "specialRest"],
      "detail": "配信の流れを次回につなげる軽い後工程。"
    }
  ]
}
```

除外をCodexから反映する場合は、次のような形式も使えます。

```json
{
  "questFeedback": [
    {
      "questId": "ai-video",
      "reason": "今はAI動画視聴より、実際にCodexへ投げる質問作りを優先したい。"
    }
  ]
}
```

重さや想定時間の調整をCodexから反映する場合は、次のような形式を使えます。

```json
{
  "questTuning": [
    {
      "questId": "diet-strength",
      "weight": "light",
      "minutes": 10,
      "frequency": "medium",
      "reason": "筋トレは10分程度にして、継続しやすさを優先したい。"
    }
  ]
}
```
